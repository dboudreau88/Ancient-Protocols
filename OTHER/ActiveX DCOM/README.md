# ActiveX DCOM

## Overview

Microsoft's Distributed Component Object Model. Remote procedure calls and object activation across networks.

## Era

1990s

## What Makes It Interesting

Remote procedure calls across networks. Component reuse and sharing. Enabled COM objects to be accessed remotely. Foundation of early Windows distributed computing.

## Revival Potential

**Low**

## Current Status

Dead. Security nightmare. Concept lives on in gRPC/REST APIs.
